BA Chat 深层链接测试页面

测试前准备：

  1. 确保 BA Chat 应用正在运行
  2. 在应用中按 Ctrl + Shift + I 打开开发者工具
  3. 切换到 Console 标签
  4. 然后点击下面的链接进行测试

1. 贴纸包链接测试

点击下面的链接应该唤醒应用并打开贴纸包预览：

点击测试贴纸链接
预期行为：
  • 应用被激活/唤醒
  • 打开贴纸包预览界面
  • 显示贴纸内容
控制台应该显示： 
handleSignalRoute: Matched signal route: artAddStickers

2. 用户名链接测试

点击下面的链接应该唤醒应用并查找联系人：

点击测试用户名链接
预期行为：
  • 应用被激活/唤醒
  • 查询用户名
  • 打开对话界面（如果找到）
  • 或显示"用户未找到"（如果不存在）
控制台应该显示： 
handleSignalRoute: Matched signal route: contactByEncryptedUsername

3. 简化贴纸链接测试

测试简化格式的贴纸链接：

点击测试简化贴纸链接
预期行为： 与测试1相同

4. 测试窗口唤醒链接

这个链接只是唤醒应用窗口：

点击测试窗口唤醒
预期行为：
  • 应用窗口被激活

如果链接无法点击：

  1. 右键点击链接 → 复制链接地址
  2. 打开命令提示符（CMD）
  3. 粘贴链接并按回车

例如：

baxs://show-window

故障排查：

  • 如果点击后没有反应 → 检查 baxs:// 协议是否已注册
  • 如果应用唤醒但没有跳转 → 查看控制台是否有 handleSignalRoute 日志
  • 如果看到错误 → 复制错误信息并发送给开发者
